## Project Overview

**Reinvención.Pro** — Premium career reinvention platform (OVO: orientación vocacional-ocupacional) powered by AI, built for Spanish-speaking professionals seeking career transitions. The master architecture document is `master_architecture_prompt.md` — it is the authoritative source for all product and technical decisions.

**Business funnel:** Free diagnostic test → AI-generated personalized report → Book 1:1 consultation → High-ticket program sale. This is NOT a subscription product.

## Commands

```bash
npm run dev      # Start development server
npm run build    # Production build
npm run start    # Start production server
npm run lint     # Run ESLint
```

No test runner is configured.

## Stack

- **Next.js 15 (App Router)** with React 19, TypeScript
- **Supabase** — Postgres + Auth (RLS enabled on all user-facing tables)
- **Vercel AI SDK** + OpenAI (`generateObject` for structured AI outputs)
- **next-intl** — i18n with server-side rendering (locales: `es` default, `en`)
- **Tailwind CSS** + Framer Motion + Radix UI / shadcn/ui
- **React Hook Form** + Zod for form validation

## Architecture

### Routing

All pages live under `app/[locale]/` — every route supports both `/es/...` and `/en/...`. Adding a new page means creating it inside `app/[locale]/`.

Key routes: home (`/`), `/diagnostico/ancla-de-carrera`, `/orientacion-vocacional`, `/terapia`, `/servicios/ingles-profesional`, `/contacto`, `/login`, `/quienes-somos`.

API routes under `app/api/`:
- `POST /api/diagnostics/analyze` — Sends career anchor test answers to OpenAI (gpt-4o) via Vercel AI SDK and returns a structured report
- `POST /api/diagnostics/save` — Persists results to `user_diagnostics` table (requires Supabase auth)
- `GET /api/auth/callback` — OAuth callback handler

### Component Architecture

- `components/layout/` — Header (with services dropdown + mobile menu), Footer, CookieBanner, `therapy-float.tsx` (floating CTA bubble)
- `components/sections/` — Landing page sections (Hero, Trust, Problem, Method, Paths, FAQ, etc.)
- `components/motion/` — Framer Motion wrappers for reusable animations
- `components/ui/` — shadcn/ui primitives (Button, Card, Dialog, etc.)

Use Server Components by default. Add `"use client"` only for interactive components (forms, context consumers, Framer Motion).

### i18n

Translations live in `messages/es.json` and `messages/en.json`. On the server, use `getTranslations()` from `next-intl/server`. On the client, use the `useTranslations()` hook. The locale is injected from `[locale]` dynamic param.

Copy tone: **neutral Spanish** ("tú", not "vos" or "usted"), empathetic and warm — not cold corporate or generic coaching.

### Supabase Clients

- `lib/supabase/server.ts` — Server-side client (cookie-based sessions, use in Server Components and API routes)
- `lib/supabase/client.ts` — Browser client (use in Client Components only)

### State Management

- `lib/cookie-context.tsx` — React Context for GDPR cookie consent (localStorage-backed, categories: essential, marketing, personalization, analytics)
- No Redux/Zustand — React Context + React Hook Form is the pattern used

### AI Integration Pattern

`app/api/diagnostics/analyze/route.ts` is the reference implementation: use Vercel AI SDK's `generateObject()` with a Zod schema for structured, validated outputs from OpenAI. The AI generates post-diagnostic reports based strictly on uploaded Schein career anchor documentation — it must not extrapolate beyond that source material.

### Styling

- CSS variables (HSL-based) defined in `app/globals.css` control the design system
- `tailwind.config.ts` extends the theme with custom typography scale, spacing tokens, and shadow utilities
- Use `cn()` from `lib/utils.ts` (clsx + tailwind-merge) for conditional class names

## Key Constraints from master_architecture_prompt.md

- The AI is a copilot — it never replaces the human psychologists
- Career routes generated by AI must pass human review before being shown to users
- No clinical/psychological diagnosis from AI
- Payments: Stripe (international) + MercadoPago (LatAm) — planned Phase 4
- Current focus (Phase 1): post-diagnostic AI report generation
